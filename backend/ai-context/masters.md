# 마스터 화면 — 거래처·은행·창고·제조사·품번·발전소

이 영역의 화면은 ERP 의 *기준 정보(reference data)* 를 등록·관리합니다. 운영 데이터(P/O·B/L·수주·출고)는 모두 여기 등록된 마스터 행을 참조하므로 **삭제 대신 비활성** 이 원칙입니다.

## 공통 규칙

- **삭제 버튼 없음 (의도적)**. 회계·세금계산서 매칭 이력 보존을 위해 마스터 행은 비활성 토글로만 숨깁니다. 비활성 마스터는 신규 PO/수주 선택지에서 자동 제외되고, 기존 거래에는 영향이 없습니다.
- **ERP코드 / 사업자번호 등 회계 연동 키는 admin 전용**. 일반 운영자는 읽기 전용. 변경 시 과거 회계 데이터와 분리될 수 있어 신중히 다뤄야 합니다.
- **이름 변경은 가급적 피할 것**. 약칭/별명 대신 사업자등록증 정식 명칭을 사용. 검색·필터·세금계산서 매칭이 모두 이름 기반.
- **신규 등록 권한은 운영팀 이상**. viewer/manager 는 조회만.

## 거래처 (`/masters/partners-v2`)

- 유형 = 공급사 / 고객사 / 양방향. 양방향은 한 회사가 우리에게 팔기도 하고 사기도 하는 경우 (드물지만 baro 테넌트에서 자주 등장).
- **결제조건은 고객/양방향에만 노출**. 공급사 결제조건은 P/O·L/C 단위로 따로 관리하므로 거래처에는 두지 않습니다.
- **활성 토글**: 비활성 거래처는 신규 등록 화면에서 안 보이지만 검색 필터에서 "비활성 포함" 으로 다시 보이게 할 수 있습니다.

## 은행 (`/masters/banks-v2`)

L/C 개설은행·수금계좌의 마스터. 은행코드는 표준 코드 (KEB하나 = 081 등) 권장. L/C 한도와 만기는 여기가 아니라 *L/C 화면* 에서 관리.

## 창고 (`/masters/warehouses-v2`)

입고·출고 시 창고 단위로 재고가 집계됩니다. 창고를 옮기면 재고는 자동 이동하지 않습니다 — *재고 조정* 화면에서 별도 처리.

## 제조사·품번·발전소

- 제조사: 패널/모듈의 제조원. 한 제조사가 여러 품번을 가짐.
- 품번: SKU. 와트, 셀 타입, HS 코드 등 — *통관 면장 자동 매칭의 핵심*.
- 발전소: 출고 목적지. 발전소별 누적 출고/매출 집계의 기준.

## 자주 묻는 질문 패턴

- "거래처 삭제는?" → 활성 토글을 OFF.
- "ERP코드 바꿔줘" → admin 권한 확인 후 안내. 회계 영향 경고.
- "비활성인데 다시 보여줘" → 화면 우상단 필터 → "비활성 포함".
