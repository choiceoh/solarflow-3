# 출고·판매·수주 화면

ERP 의 *판매 측* 흐름. 거래처가 우리에게 주문(수주) → 우리가 발송(출고) → 매출 인식(판매) → 입금 회수(수금) 순서. 재고가 차감되는 시점은 **출고 등록 시**, 매출이 인식되는 시점은 **세금계산서 발행 시**.

## 공통 규칙

- **출고 등록은 즉시 재고 차감**. 잘못 등록한 경우 *출고 취소* 흐름(draft → pending → cancelled, 3 단계)으로 정상화. 직접 DB 삭제 금지.
- **수주 잔량 자동 추적**. 수주 100장 중 60장 출고 시 status=partial, 100장 모두 출고 시 completed.
- **세금계산서는 출고와 별도**. 출고 후 세금계산서 미발행 건이 대시보드에 알림으로 노출됨 — 월말 마감 전 일괄 발행이 흔한 패턴.
- **그룹 거래(group_trade=true)** 는 같은 그룹사 간 이동. 사외 매출과 분리해 집계됨.

## 화면별

- **수주 (`/orders`)**: 거래처별 판매 주문서. 상태=받음/부분출고/완료/취소.
- **출고 등록 (`/outbound`)**: 수량·현장(발전소)·용도(태양광/풍력/ESS) 별로 기록. 발전소는 마스터에서 사전 등록되어 있어야 함.
- **출고 취소**: 활성 출고를 바로 cancelled 로 못 바꿈 — *취소 신청* (cancel_pending) → manager 승인 → cancelled.
- **수금 매칭 (`/receipts`)**: 입금액과 미수금 자동 매칭. 매칭 확정은 manager 권한.

## 권한 주의

- **viewer / staff**: 판매단가(unit_price_wp), 마진율, 이익액 모두 차단. "이번 달 매출 얼마?" 같은 질문에 답하지 말 것.
- **manager**: 매출 세금계산서 발행, 수금 매칭 확정, 출고 취소 승인 권한.
- **operator / admin**: 단가·이익 포함 모든 정보 조회 가능.

## 자주 묻는 질문 패턴

- "이번 달 출고 합계는?" → 기간 필터(이번 달 1일 ~ 오늘)로 outbounds SUM. 단가 노출은 권한 확인.
- "미출고 수주는?" → orders.status=partial 또는 received.
- "취소된 수주는?" → orders.status=cancelled. 취소 사유는 별도 메모로 기록.
- "세금계산서 미발행 출고는?" → outbounds 중 sales.tax_invoice_date IS NULL.
- "거래처별 미수금?" → 수금 매칭 화면에서 고객 선택 시 자동 표시.

## 연결

- **inbound** B/L 완료 → 재고 증가 → 그제서야 outbound 가능. 재고 부족 시 출고 등록 자체가 막힘.
- **수금** → LC 한도 복원 (수입 측 banking 도메인). 즉 우리가 받는 돈이 다음 LC 개설 여력으로 직결.
