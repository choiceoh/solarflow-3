# OCR 자동입력 정리와 다음 작업

## 현재까지 완료

### 문서 OCR 기반
- `/api/v1/ocr/extract`가 PDF/이미지 파일을 받아 OCR 원문과 구조화 후보를 반환한다.
- `document_type=customs_declaration`일 때 수입필증/면장 후보 필드를 별도로 파싱한다.
- OCR 결과는 즉시 DB에 저장하지 않고, 사용자가 확인한 뒤 입력칸에 반영한다.

### 면장/수입필증 자동채움
- 인식 대상 필드:
  - B/L(AWB) 번호
  - 입항일
  - 수입자
  - 운송주선인
  - 무역거래처
  - 국내도착항
  - 모델/규격
  - 수량
  - 단가
  - 금액(USD)
  - CIF 원화금액
  - 환율
- B/L 입고 폼 반영 대상:
  - B/L 번호
  - 실제입항일
  - 포워더
  - 항구
  - 입고품목
  - 면장 CIF 원화금액
  - 환율
- 수입자, 무역거래처, HS코드, 세관 등 현재 B/L 본문에 직접 저장하지 않는 값은 참고값으로 보여준다.

### 사용자 흐름
- 입고관리 화면에 PDF/사진을 드래그하면 입고등록 창이 열린다.
- 입고 구분은 해외직수입으로 자동 선택된다.
- OCR 입력값 확인창이 먼저 뜬다.
- 사용자가 `확인 후 입력칸에 반영`을 눌러야 폼 값이 채워진다.
- 입고등록 폼 안에서도 PDF/사진 선택 또는 드래그 업로드가 가능하다.

## 알려진 제한

- 품목 자동채움은 품번코드/품번명, 출력(Wp), 모델 토큰으로 후보를 잡고 확인창에서 수동 선택할 수 있지만, 실사용 샘플별 정확도 측정은 더 필요하다.
- 수입자와 무역거래처는 PO 연결값이 없을 때만 법인/제조사 후보로 반영하며, PO 연결값이 있으면 참고값으로만 보여준다.
- OCR 신뢰도별 필드 색상, 낮은 신뢰도 경고, 원본 좌표 하이라이트는 아직 없다.
- 여러 제조사·포워더·세관 양식 샘플이 충분히 쌓이기 전까지 좌표 파서는 샘플 중심이다.
- OCR 후보를 첨부파일 원본과 함께 B/L 상세에 자동 보관하는 흐름은 아직 없다.

## 다음 작업

1. 실사용 샘플 5~10건 검증
   - 탑솔라 수입필증 외 다른 날짜/포워더/세관 파일로 필드별 인식률을 확인한다.
   - 잘못 읽은 필드를 기록해 파서 보정 기준을 만든다.

2. 품목 매칭 고도화
   - 실사용 샘플 결과로 모델 토큰/제조사 별칭 규칙을 보정한다.
   - 수동 선택한 품번과 OCR 원문을 추후 학습용 로그로 남길지 판단한다.

3. 거래처/법인 자동 후보
   - 수입자와 무역거래처 후보가 빗나간 샘플을 모아 별칭 테이블 전환 여부를 판단한다.
   - 자동 확정하지 않고 확인창에서 후보로 제시하는 원칙을 유지한다.

4. 확인창 품질 개선
   - 신뢰도 낮은 값은 경고 색상으로 표시한다.
   - 원본 OCR 텍스트와 후보값을 나란히 비교할 수 있게 한다.
   - 품목 후보를 수정하거나 제외할 수 있게 한다.

5. 저장 후 추적성
   - 사용한 PDF/사진을 B/L 첨부파일로 함께 보관한다.
   - 반영한 OCR 후보 요약을 메모 또는 감사 로그에 남긴다.

6. 회귀 검증
   - 샘플 면장 텍스트 기반 parser 테스트를 늘린다.
   - 입고관리 드래그 → 입고등록 → OCR 확인창 → 반영 흐름을 E2E smoke에 추가한다.
