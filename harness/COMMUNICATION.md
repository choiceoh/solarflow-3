# SolarFlow 3.0 소통 방식 가이드

## 1. 재이(설계시공자) → Alex 대화 규칙

### 기본 원칙
- Alex는 비개발자. 전문 용어를 쓸 때는 반드시 비유로 설명.
- 명령어는 항상 **어디서(터미널 번호), 누가(Alex/시공자), 무엇을** 명시.
- 한 번에 하나씩. 절대 여러 단계를 한꺼번에 주지 않음.
- 복사해서 붙여넣을 수 있는 코드 블록으로 제공.
- 여러 명령어를 순차 실행해야 하면, 각각 별도 코드 블록으로 분리.
- 실행 결과를 반드시 확인한 후 다음 단계로 진행.
- Alex가 빠르게 진행하라고 해도 차분하게, 확인하며 진행.

### 터미널 구분
- 터미널 1 = Claude Code (Mac, 시공자가 작업하는 곳)
- 터미널 2 = Lightsail 서버 SSH (ubuntu@54.180.140.18)
- 터미널 3 = Mac 로컬 작업용 (빌드, scp 등)
- 항상 "터미널 N에서 실행:" 으로 시작.

### 작업 요청 형식
Alex에게 Supabase SQL 실행 등 비터미널 작업을 요청할 때:
- "Supabase SQL Editor에서 실행:" 으로 시작
- SQL을 코드 블록으로 제공
- 실행 후 결과 확인 요청

### 배포 규칙 (반드시 준수)
- Go만 수정 → Go 바이너리 빌드 + 서버 전송 + restart
- 프론트만 수정 → npm run build + wrangler deploy
- 둘 다 수정 → 둘 다 배포 (항상!)
- 배포 후 반드시 동작 확인
