# SolarFlow 3.0 설계서 v4 — 시공 기준 문서

이 문서 = 시공 도면. 이 문서에 없는 변경 금지.
시공자가 판단할 것 없음. 모든 화면/입력폼/CRUD 완전 정의.

## 메뉴 구조

[업무흐름]
  1. 발주관리 — PO 등록, 입고품목, T/T 송금
  2. LC 관리 — PO에서 내려옴, LC 개설/수정/삭제
  3. 입고관리 — LC에서 내려옴, BL 등록, 면장환율 포함
  4. 재고현황 — 가용재고(첫탭), 미착품, 재고상세, 수급전망
  5. 출고/판매 — 상품판매, 공사사용, FIFO 매칭
  6. 수주/수금 — 수주, 수금, 매칭

[현황/분석]
  대시보드, LC 한도/만기, 매출/이익분석(거래처/월/분기/반기/연간)

[도구]
  마스터관리(6종), 검색, 메모, 결재안, 엑셀, 설정

삭제된 메뉴: 면장/원가 -> BL 상세 [면장/원가] 탭으로 이동

## 설계 결정

D-087: PO 기준 전체 데이터 자동 연결
D-088: BL outbound_status 별도 컬럼 (입고상태와 분리)
D-089: FIFO 원가 매칭 Rust 계산, Go 저장 (D-025 패턴)
D-090: 스페어 출고 원가 판매원가에 포함
D-091: 메뉴 업무흐름 기반 재배치
D-092: PO 변경은 부분 수정 + 신규 PO
D-093: 입고등록은 LC 선택이 기점 (국내구매는 PO 직접)
D-094: PO 법인과 LC 법인 다를 수 있음. PO는 그룹 전체, LC에서 법인 결정
D-095: BL수량 > LC물량 = 차단. LC amend 필요. BL > PO = 경고만
D-096: 면장/원가 별도 메뉴 삭제. 면장환율을 입고등록에 통합. 부대비용은 BL상세 탭
D-097: 재고현황 첫 탭 = 가용재고
D-098: 매출/이익분석 거래처별 월별 분기별 반기별 연간

## 용어 변경 (전체 적용)
- 라인아이템 -> 입고품목 (사용자에게 보이는 모든 곳)
- 기본정보 -> 종합정보 (PO 상세)

## PO/LC 선택 드롭다운 표시 형식
PO: PO번호 | 제조사 | X.XMW | YYYY-MM
LC: LC번호 | 제조사 | 은행 | 금액 | X.XMW

## 무상출고 vs 스페어 (치명적 구분)
엑셀 상품판매 = sale (본품, 이익분석 대상)
엑셀 상품판매 내 스페어 = sale, item_type=spare (매출0, 원가포함)
엑셀 무상출고 = construction (공사사용, 이익분석 제외)

## 발주관리 상세

### PO 목록
컬럼: PO번호, 제조사(한글), 계약유형, 계약일, Incoterms, 총수량EA, 총MW, 상태
필터: 상태, 제조사, 유형
버튼: + 새로 등록

### PO 등록 폼 (다이얼로그)
구매법인*(드롭다운), PO번호*(텍스트,20자), 제조사*(드롭다운), 계약유형*(스팟/프레임)
계약일*(달력+직접입력), 선적조건*(CIF/FOB), BAF/CAF체크, 환율, 상태*(예정/계약완료만 수동)

발주품목: 품번*(드롭다운->모델명,규격,크기mm 자동표시), 수량EA*, ->MW(자동), 단가센트/Wp*, 총액USD(자동), 용량MW(자동), 본품/스페어, 유상/무상
수량입력->MW자동, MW입력->수량자동
합계: 총수량, 총MW, 총금액USD, KRW(환율입력시)

결제조건: 계약금 있음/없음, T/T, 비율%, 금액자동, 분할추가(최대5), 잔금LC 일수
메모

저장검증: 필수필드, 품목1건이상, 수량양수, 단가양수(유상), PO번호중복불가

### PO 수정
등록과 동일 다이얼로그. 기존 데이터 전부 로드 필수(발주품목 포함).
발주품목: 기존 UPDATE + 신규 INSERT + 삭제 DELETE (diff CRUD)
PO번호 수정 불가.

### PO 삭제
BL/LC/TT 연결 있으면 차단. 없으면 삭제 가능.

### PO 상세
탭 4개: [종합정보] [입고품목] [LC현황] [입고현황]
우상단: 상태배지, 수정, 삭제, 입고등록

[종합정보]
기본필드 + T/T 납부현황(기납부/잔여) + LC 개설현황(기개설/미개설잔액) + 진행률바 + T/T이력테이블
진행률: 계약MW, LC개설MW(%), 선적MW(%,BL등록기준), 입고완료MW(%,completed기준)

[입고품목]
품번, 모델명, 규격, 크기mm, 수량EA, MW, 단가센트/Wp, 총액USD, 구분, 유무상
+ 추가, 행별 수정

[LC현황]
PO총액 대비 현황: PO총액, TT납부, LC개설, LC미개설
LC 목록: LC번호, 은행, 개설법인, 개설일, 금액, MW, 만기일, 상태

[입고현황]
BL번호, LC번호, 입고일, 수량EA, MW, 면장환율, 입고상태, 출고상태

### PO 상태 자동 전환
예정(draft): 등록 시 기본
계약완료(contracted): 수동 전환
선적중(shipping): BL 1건 이상 등록되면 자동
완료(completed): PO 전량 입고완료되면 자동
Go 백엔드: BL 생성/수정 시 syncPOStatus 호출

### T/T 등록 폼
PO 상세 종합정보에서 + T/T 등록 버튼
PO 현황 표시(계약금, 기납부, 잔여)
송금일(달력+직접), 금액USD*, 환율, 원화(자동), 목적*(계약금1차~기타), 은행, 상태*(예정/완료), 메모
T/T 수정: 행 수정아이콘 -> 동일폼, 기존데이터 로드
T/T 삭제: 확인 후 DELETE

## LC 관리 상세

### LC 목록
컬럼: LC번호, PO번호, 제조사, 은행, 개설법인, 개설일, 금액USD, MW, 만기일, 상태
필터: 상태, 은행, 법인
버튼: + LC 개설

### LC 개설 폼 (핵심 신규 화면)
PO 선택*(드롭다운, 표시: PO번호|제조사|MW|YYYY-MM)

PO 선택 시 자동 표시(수정불가):
PO 계약총액, T/T 기납부, LC 기개설, LC 미개설잔액

LC번호, 개설은행*(드롭다운), 개설법인*(드롭다운, PO법인과 다를수있음 D-094)
개설일(달력+직접), 개설금액USD*, 대상수량EA, 대상MW(자동)
Usance일수, Usance유형(Buyers/Shippers), 만기일(자동=개설일+Usance)
상태*, 메모

개설 후 잔여 실시간 표시

검증: 개설금액 > 미개설잔액 -> 경고(저장허용, amend대비)

### LC 수정
등록과 동일, 기존데이터 로드. 금액/수량/만기 수정 가능.
PO 범위 초과시 경고.

### LC 삭제
연결 BL 있으면 차단.

## 입고관리 상세

### BL 목록
컬럼: BL번호, PO번호, LC번호, 입고유형, 제조사, 법인, 규격, 크기mm, 수량EA, MW, 입고일, 면장환율, 입고상태, 출고상태
필터: 유형, 상태, 제조사
버튼: + 새로등록, 양식다운로드, 업로드, 아마란스입고

### BL 등록 폼 (LC에서 데이터 내려옴 D-093)

입고구분*: 해외직수입 / 국내구매 / 그룹내구매

[해외직수입]
LC 선택*(드롭다운, 표시: LC번호|제조사|은행|금액|MW)
LC 선택시 자동표시: PO정보, LC개설금액, 기선적, LC잔여
자동채움(수정불가): 제조사, 법인, 통화(USD)
자동채움(수정가능): Incoterms, BAF/CAF

[국내구매]
PO 선택*(드롭다운, LC 없으므로 PO 직접)
자동채움: 제조사, 법인, 통화(KRW)

[공통입력]
BL번호*, ETD(달력+직접), ETA(달력+직접), 실제입항일*(달력+직접)
면장환율(비워두면 미착품), 항구(드롭다운), 포워더, Invoice No.
입고창고(드롭다운), 메모

면장환율 안내: 비워두면 미착품으로 등록. 입고완료 전환시 필수.

PO 발주품목에서 입고:
품번, 모델명, 규격, 크기mm, 계약EA, 기입고EA, 잔여EA, 이번입고EA 입력란, + 버튼
+ 클릭시 입고품목에 추가 (품번, 단가, 구분, 유무상, po_line_id 자동복사)

입고품목:
품번, 모델명, 규격, 크기mm, 수량EA, 구분(본품/스페어), 유무상, 단가센트/Wp, 인보이스USD(자동), 인보이스KRW(자동), 용량kW(자동)
+ 수동추가 버튼

자동계산:
인보이스USD = 수량 x 단가(센트) / 100 x spec_wp
인보이스KRW = 인보이스USD x 면장환율
용량kW = 수량 x wattage_kw

저장검증:
BL번호 필수, 중복불가
입고품목 1건이상
이번입고 > LC잔여 = 차단 (D-095)
이번입고 > PO잔여 = 경고 (저장허용)
면장환율 비어있으면 status=shipping(미착품)

### BL 수정
등록과 동일, 기존데이터 로드
면장환율 추가입력 -> 미착품에서 입고완료 전환 가능
출고된 BL 수량축소 -> 출고수량보다 적으면 차단

### BL 삭제
출고 연결 있으면 차단

### BL 상세
탭: [기본정보] [입고품목] [면장/원가] [출고추적]

[기본정보] PO번호(클릭이동), LC번호(클릭이동), BL번호, 제조사, 법인, 입고유형, 면장환율, ETD/ETA/입항일, 항구, 포워더, 창고, Invoice, 입고상태, 출고상태

[입고품목] 품번, 모델명, 규격, 크기mm, 수량, MW, 구분, 유무상, 단가, 인보이스USD, CIF원/Wp

[면장/원가] (D-096 면장메뉴 통합)
면장정보: 면장번호, 수입신고일, 세관, HS코드, 입항일, 반출일
원가: 면장환율, CIF원화총액(자동), CIF Wp단가(자동)=회계원가
부대비용: CRUD (11유형), 합계
Landed Cost: CIF+부대비용배분+관세 = Landed Wp단가(실무원가), 저장버튼
환율비교: 발주환율(PO) vs 면장환율(BL) vs 송금환율(TT) 3단계

[출고추적] (Round 4에서 구현)
입고수량, 출고합계, 잔여, 출고상태배지
출고건 목록: 출고일, 거래처/현장, 유형, 본품EA, 스페어EA, 판매가, 원가, 이익

### 면장환율과 상태 전환 (D-095 보완)
Go 백엔드: BL status를 completed로 변경시 exchange_rate NULL이면 에러 반환
프론트: 입고완료 선택시 면장환율 비어있으면 빨간테두리 + 메시지 + 저장차단

## 재고현황 상세

탭 4개: [가용재고] [미착품] [재고상세] [수급전망]

[가용재고] (첫번째, D-097)
요약카드: 물리적재고MW, 가용재고MW, 미착품MW, 총확보량MW
테이블: 제조사, 모델명, 규격, 크기mm, 물리EA, 물리MW, 예약, 배정, 가용EA, 가용MW
같은 Wp라도 크기 다르면 별도 행
집계공식: 물리적=입고완료-출고, 예약=stock수주잔량, 배정=stock공사잔량, 가용=물리-예약-배정

[미착품] BL번호, PO번호, 제조사, 모델명, 규격, 크기mm, 수량, MW, ETD, ETA, 상태

[재고상세] BL별 드릴다운: BL번호, PO, 입고일, 면장환율, CIF원가, Landed원가, 입고EA, 출고EA, 잔여EA, 출고상태

[수급전망] 기존 유지

## 출고/판매 상세

### 출고 등록 폼
출고구분*(상품판매/공사사용/그룹내거래)
출고일*(달력+직접), 법인*, 출고창고*
