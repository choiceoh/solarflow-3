# SolarFlow 역할 정의서

## 역할 구조
## 시공자 (Builder)
- **하는 일**: TASK 파일 기준으로 코드 작성, 빌드/vet 실행, 체크리스트 보고
- **하지 않는 일**: 설계 방향 임의 변경 (Go만으로 충분하다고 Rust 제거 금지)
- **TASK 작성**: 시공자가 초안 작성 → 감리자 검토 → 승인 후 작업
- **보고 양식**: CHECKLIST_TEMPLATE.md 따를 것
- **코드 제출**: 전체 코드(cat) + 빌드 결과 + 체크리스트

## 감리자 (Inspector)
- **하는 일**: 설계문서 대비 코드 대조, 패턴 일관성 검증, 빠진 필드 지적, 방향 판단
- **하지 않는 일**: TASK 직접 작성 (시공자가 작성 → 감리자 검토), 코드 직접 작성
- **판정 기준**: 
  - 10/10: 설계문서 100% 반영 + 잠재 문제 없음
  - 9/10: 반영 완료, 사소한 잠재 위험 1~2건
  - 8/10: 동작하지만 구조적 개선 필요
  - 7/10 이하: 불합격, 재작성

## Alex (Project Owner)
- **하는 일**: 실무 판단 (이 설계가 현장에서 쓸 수 있는가?), 시공자↔감리자 결과 전달, 최종 승인
- **하지 않는 일**: 코드 직접 수정 (시공자에게 지시)
